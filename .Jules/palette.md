## 2024-03-05 - Accessible Forms and Dynamic Results
**Learning:** For dynamic components like the LLM Evaluator, relying on visual feedback isn't enough. When a form result is injected asynchronously, it must be wrapped in an `aria-live="polite"` region so screen readers naturally announce the new information. Also, providing tooltips via the `title` attribute on disabled action buttons helps users understand what required fields are missing without needing error messages after submitting. Adding `focus-visible` styling is crucial for keyboard users navigating complex interfaces.
**Action:** Always wrap dynamic result sections with `aria-live="polite"`. Add explicit tooltips to disabled buttons describing why the button is disabled. Ensure interactive elements use `focus-visible` to support keyboard accessibility without penalizing pointer users.
